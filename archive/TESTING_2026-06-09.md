Test everything on the **DEV** builds: **"Skrift Dev"** with the **inverted (pink) icon**, on your
phone and Mac. Dev has its own data + the test vault — your real "Skrift" memos/vault are untouched.
Work through the chapters, check each box, record yourself. When it all passes we **promote to prod**

## 0. Confirm you're in DEV (both devices)
- [ ] Mac: the app you're testing is **"Skrift Dev"** with the **inverted icon** (not "Skrift").
- [ ] Phone: the app is **"Skrift Dev"** with the inverted icon.
> If the name/icon match prod, you opened the wrong one.

## 1. Record screen (phone)
- [ ] **Model-loading placeholder** — open Skrift Dev fresh and **start recording immediately** (before it's "ready"). The caption area should say *"Model loading — your words appear once it's ready"*, then fill in + catch up. (Before: it just looked empty/broken.)
- [ ] **Auto-scroll + scroll-back** — talk for a while. Newest text should stay visible automatically. Swipe **up** to re-read older text; scroll back to the bottom and it should resume sticking to the newest line.
- [ ] **Confidence colour** — while talking, the **last few words are lighter** (still being decided) and older text is **solid**. *(This is an approximation — the model doesn't tell us which words are final. Judge if it feels useful; I can tweak how many words / the shade.)*
- [ ] **Photo marker** — tap the photo button mid-recording and take a photo. A tinted **`[photo 1]`** should appear **inline** in the live text where you took it.
- [ ] **AirPods robustness** — record with AirPods in, then **pull one out** mid-recording. Recording should **keep going** (not die) and show a brief amber notice. *(Heads-up: double-check the part you said right after pulling them is in the transcript — that's the one edge I couldn't verify without the hardware.)*

## 2. Memos list (phone)
- [ ] **Title on the row** — give a memo a **title** (open it, edit the title), go back to the list. The row should now show your **title** on top, with the transcript as a smaller second line. (Untitled memos look the same as before.)
- [ ] **No "Waiting" on phone-only memos** — a memo left at **significance 0** (the default) should show **no sync pill** at all on the list. Only memos you bump above 0 show "Waiting"/"Synced".

## 3. Memo detail (phone)
- [ ] **Add-recording "+" button** — open a memo. There's now a **`+` button top-right**. Tap it → record a bit more → it should **append** the new audio + text to that memo.
- [ ] **Keyboard dismiss on scroll** — open a memo, tap into the transcript so the keyboard shows, then **drag/scroll the page**. The keyboard should **dismiss** (so you can read with the audio bar open).

## 4. Video import (phone + Mac)
- [ ] **Phone** — **share a video** into Skrift Dev (Share sheet → Skrift Dev) **or** import via the Photos picker. It should become a memo: audio **transcribed**, a **frame thumbnail** attached, and the **date = when the video was filmed** (not now).
- [ ] **Mac** — drag a **video file** onto the Skrift Dev sidebar. It should extract the audio and process it like a normal note.

## 5. Desktop (Mac)
- [ ] **Liquid Glass transport bar** — open a note that has audio in Skrift Dev. The bottom **play/transport bar** should read as a **floating glass capsule**. *(Judge live — if you want it more see-through I flip one word, `.regular`→`.clear`.)*
