Build notes — locked decisions (2026-06-25)

  • Inline display = [[Canonical|spoken]]. Linked names read as the SPOKEN word ("Jack", "Hendri") for natural prose; the link targets the full canonical (shown in the sheet header + the exported [[Hendri van Niekerk|Hendri]]). The "first-mention-canonical / rest-short" rule is for conversation speaker headers, NOT inline prose.
  • One link per person, first mention only. Later mentions + pronouns ("him") are plain body text — no grey "repeat" tier.
  • Three tiers. LINKED #9d8ff7 solid; SUGGESTED tan #bda481 + dotted #c4a982 (single candidate, one tap); AMBIGUOUS plain text + accent bg + purple dotted + ? (multiple candidates).
  • iOS bottom action sheet (the native confirmationDialog idiom) WITH a separate Cancel card — never an anchored caret-popover. Scrim or Cancel dismisses.
  • "Change person…" only when AMBIGUOUS. A name only one person has (Hendri) gets Unlink + Open; an ambiguous one (Jack → Hutton/Tanner) also gets Change person…
  • "Keep as plain text" is reversible. The name stays a faint dotted token, re-tappable to link later (or via the chip bar). Unlink → plain + an Undo toast.
  • "Open … person card" → the on-device Names editor (state 5). Jumping to the Obsidian note is a deferred "later" idea.
  • People in this note (state 4) = link / re-link surface: candidate chips link a first mention, on-chips unlink, "Someone else…" → editor.
