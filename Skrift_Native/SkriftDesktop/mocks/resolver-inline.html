My recommendation: A. It reads as the note (prose stays prose), gives full-paragraph context, and maps cleanly onto the Mac text view (each ambiguous range becomes a clickable link → an NSPopover anchored at the word). Per-occurrence is automatic, which kills the old card’s biggest gap. The slim top banner stays as a “2 left · jump to next” affordance so you never lose track. B is the most discoverable but noisiest; C is best only if notes routinely have many names. All three apply through the existing order-based Sanitiser.applyResolvedOccurrences, so the engine work is done — this is the front-end.
