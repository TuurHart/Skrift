The Unrated shelf — the notes the Mac isn’t processing

The Mac only pulls a memo into the queue when you’ve rated it (MemoCloudIngest skips significance 0). Everything else syncs down and sits there invisible — which is how a note recorded on the phone can be “missing” on the Mac. This adds a third shelf next to Fading and Recently Deleted, using the same idiom: one row, one count, one click. Rating from here does double duty — it queues the note for processing and rescues it from fading, because significance > 0 is both the process gate and the first touch signal in MemoLifecycle.isTouched.

Rating a note flags it for processing and stops it fading — one action, both effects. Notes you never rate fade after 30 days and clear themselves.

The fourth row is the honesty case. That note is unrated but tagged, so isTouched is already true and it will never fade — showing it a “fades in N days” countdown would be false. It reads “tagged · won’t fade” instead. Same rule as the Connections panel’s first-mention flag: the shelf may not claim a deadline it can’t keep.

Note the verb is “starts fading”, not “fades”. The Fading shelf already uses “fades in N days” for the 60-day move to Recently Deleted (daysUntilSweep); this shelf counts to the 30-day moment a note lands on that shelf. Two different events — they must not share a phrase.

decisions this mock is asking you to lock
  • Third shelf, not a new tab. Unrated joins Fading + Recently Deleted in the Review rail, same row shape, same count, same click-to-open. Nothing auto-opens.
  • Glyph ○ — an empty circle, echoing the unfilled significance circles. Fading keeps 🍂, Deleted keeps 🗑.
  • Membership = significance == 0, transcribed, not trashed — literally “what the Mac’s gate is skipping”, not a second fading rule.
  • Fade countdown only when it’s true. An unrated-but-touched note (tags, title, lock, reminder…) shows “won’t fade” instead of a deadline it doesn’t have.
  • “Starts fading in N days” — deliberately NOT the Fading shelf’s “fades in N days”, which already means the 60-day trash move. Different event, different phrase.
  • “Process all N” in the header, mirroring the Queue’s own Process button — the bulk escape hatch, replacing the buried “Process every synced memo” setting as the thing you actually reach for.
  • OPEN — the row action: A (one “Process” button → 0.1) or B (three tier chips → 0.3 / 0.6 / 0.9). Recommendation: A.
