Build notes

• Today's flaws this redesigns. The shipped OnboardingView.swift:28 tagline reads "Record voice memos, transcribed on-device, synced to your Mac" and the very first run shows a Pair-a-Mac step with a live discovery.start() Bonjour scan (:39-47, 145-150) sitting between media and model permissions — a Mac-less buyer is told they need hardware they don't have. SettingsView.swift:46-76 opens with a "Mac" section first. Both communicate "this is a Mac accessory," which is wrong for a $0.69 standalone app.

• State 1 (value prop) leads with the on-device promise — "Record anywhere. Transcribed on your iPhone." No Mac, no accounts, no jargon. The mic hero matches the app's accent (#7E74F0); the 3-dot pager sets a 3-step expectation.

• State 2 (private + engine) makes "nothing leaves your phone" the headline (the privacy story IS the no-cloud-LLM tradeoff from the plan's locked decisions). The one-time engine download reuses the existing ModelLoadStatus.downloadProgress + TranscriptionService.ensureLoaded() seam already wired in onboarding — only the copy and ~600 MB / Wi-Fi framing change (today it says "494 MB"; bump to the real bundled total). Non-blocking: "Continue when ready", same best-effort pattern as today.

• State 3 (sync) promotes the NET-NEW Phase-1 CloudKit toggle ("Sync across my devices", default on, degrades to local if iCloud is off) to the hero, and demotes the Mac to a small "Connect a Mac (optional) ›" link below — clearly secondary, no auto Bonjour scan on this screen. The scan only starts if the user taps through (gate discovery.start() behind the opt-in, not onAppear).

• Settings IA (state 4) reorders to Capture · Playback · Library (Names & voices · Models · Export) · Sync · Appearance · Feedback · About. The Mac is no longer a top-level section — it lives as an Advanced → Connect a Mac row inside Sync. New rows vs today: Export (Phase 2 — default format / vault folder / include-audio), iCloud toggle (Phase 1), and the alias-management entry folds into Names & voices (Phase 2, mirroring the Mac).

• Code seams (service layer unchanged). Everything gates on existing config: MacConnection.load() != nil or a stored opt-in flag. MacTransportFactory.make() (Services/Sync/MacTransport.swift:82-97) already returns Disconnected when unpaired — no transport change. UI gating only: OnboardingView.swift:28/39-47/145-150 (tagline + pair card + scan), SettingsView.swift:46-76 (Mac section → Advanced row), MemosListView.swift:294-303 sync button + :338-339 "Pair a Mac" banner hidden unless paired.

• Real grounding. Custom words show real names Tuur, Roksana; the engine is Parakeet on the ANE; Models reads the existing read-only inventory (ModelInventory.swift), extended to managed in Phase 4. Out of scope here: the Models picker UI and the Export action sheet get their own mocks (plan §"UI surfaces needing mocks").

• Decisions to weigh in on: (a) sync screen copy — "iCloud" vs the friendlier "across my devices" as the primary label; (b) whether "Connect a Mac" lives under an Advanced row in Sync (shown) or as its own dimmed bottom section; (c) should State 2 hard-require the engine before "Start" (no transcription without it) or let users proceed and download lazily on first record.
