Semantics flagged for sign-off: “Unlink this mention” restores the alias as spoken (“Nick”, not “Nick Jansen”) and only edits this note. “Unlink all mentions in this note” also marks the person plain-in-this-note so a later re-process won't re-link them here. Neither touches the Names database — removing a person entirely stays in Settings → Names. A third scope (“never link Nick in any note”) was deliberately left out: that's a Names-level rule, not a body click.
