Build notes

The data already exists. Every quote-capture is already a memo carrying bookTitle / bookAuthor / bookChapter (the C2 contract in MemoMetadata / UploadPayload) plus the voice ramble as the body — captured via MergedCaptureView → RecordView(appendTo:). Significance is the existing 10-circle scale (SignificanceCircles.swift / SignificanceScale). Source glyphs (mic / video / book-quote / link) and the place/weather/time context chips are the same taxonomy already drawn in MemoDisplay + MemoDetailView:343-357. So Phase 6 is almost entirely a new read-surface over the store we already have — no new capture, no new sync field.

What's net-new

  • Highlights view — a third top segment (Memos · Highlights · People) backed by a SwiftData query for source == .audiobookQuote || significance > 0, grouped by bookTitle (then by source for the voice/video highlights). The quote renders with the existing accent-left-bar treatment (CaptureQuote / CaptureQuoteFrame) and the ramble shows beneath as the annotation, lifting the same italic-quote + mic-prefix layout from TranscriptBodyView. Searchable + taggable via the existing MemoFilter machinery — no new index.
  • Daily Review resurfacing — a small on-device spaced-repetition scheduler (last-seen + interval per highlight, persisted next to the memo; SM-2-lite, no ML). Surfaces N/day, one card at a time, "captured N ago" computed from recordedAt. Pure presentation — nothing leaves the phone (the reassurance copy is load-bearing for the privacy story and the App-Store label). This is the north-star "watch my thinking evolve" hook in embryo.
  • Quote card via ImageRenderer — a SwiftUI view rendered to a 1080×1080 PNG. The warm tint is UIImage.averageColor on the cover (we already compute exactly this for the player header tint in AudiobookPlayerView), darkened to the cover's hue — same trick, new consumer. Wordmark + author/book/chapter from the memo's book metadata. Handed straight to a ShareLink / activity sheet (the same seam Phase 2 adds to MemoDetailView:128-133).

Real code seams this touches

  • Per-source Markdown export folds into Phase 2. Readwise-style per-book highlight .md (append-update, not duplicate) is just MemoExporter + ObsidianPublisher grouped by book — one file per source under <vault>/Skrift/Audiobook Quotes/, sticky exportRelativePath + exportedContentHash so re-export overwrites its own file (no conflict copy). No new export engine.
  • Quote bytes stay protected. The quote text comes from the same quote-protected body the pipeline already guards (QuoteProtection); the card + the highlight feed render the quote verbatim, never the polished ramble's words inside the > block.
  • CloudKit-clean. The review schedule fields are additive/defaulted on Memo (lastReviewedAt / reviewInterval) → mirror to CloudKit with no migration, like the Phase-1/2 export-identity fields.
  • No Mac required. The whole surface reads the on-device store; if a Mac is paired it changes nothing here (Highlights is a sink-agnostic read).

Design decisions to weigh in on

  • Nav placement: mock shows BOTH a top segmented control (Memos · Highlights · People) and a 4-item bottom tab bar (Memos · Highlights · People · Books). Pick one — running both is redundant. My lean: bottom tab bar as the app's primary IA (it scales to Books + future People/Journal), drop the segmented control. Flagged for your call.
  • Daily Review default count + cadence: mocked at 3/day. Is daily resurfacing opt-in (a Settings toggle + notification) or always-on with a soft badge? Readwise pushes daily; Apple-Store-friendly default might be quieter.
  • Quote-card tint source: book quotes get the cover-derived warm tint (shown). What tints a voice-memo highlight card with no cover — the accent purple, or a tint derived from the place/weather chip? Mocked the book card since it's the marketing hero.
