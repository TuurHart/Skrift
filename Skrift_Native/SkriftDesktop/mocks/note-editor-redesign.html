Skrift — note editor re-foundation (fork B): B1 vs B2

The note body moves onto a natively-scrolling text view — the scroll view is the editor, so selection-drag autoscroll, caret-follow, magnifier, undo and find-in-note come from UIKit instead of patches. Everything below the title (chips · tags · importance · summary · quote block) scrolls with the note in both options. The one decision this mock asks for: where the title lives.

B1 · NOTES-STYLE

Title is the first line of the note

Exactly Apple Notes: the big title scrolls away with the content; once it's off-screen the nav bar picks it up. Maximum text on screen while editing.

What this mock decides

  • B1 vs B2 — title in the flow (Notes-style, nav picks it up on scroll) vs a pinned compact title row. Everything else on this page is identical between the two.
  • Accessory bar contents — proposed: undo · redo · find · photo-at-caret · Done. Say what to add/drop (candidates: ▶ play-from-caret, ＋ append recording).
  • "Save highlight" in the edit menu — in v1 or later?

Identical in both (the re-foundation itself)

  • The body is ONE scrolling text view; chips / tags / importance / summary / capture-quote live in a header that scrolls with the text. No more non-scrolling-textview-inside-ScrollView.
  • Selection autoscroll, caret-follow, magnifier, smart paste, shake/3-finger undo — native, un-patched.
  • Karaoke = attribute painting on the same view (play/pause no longer swaps renderers): highlight moves at word rate instead of re-rendering at 20 Hz, tap-a-word-to-seek = character hit test, inline photos keep their exact size and position when playback starts.
  • Name tiers (linked / suggested / ambiguous) render and tap exactly as today.
  • Typing gets cheap: saves debounce (~1 s idle + on Done/close), the name-scan memoizes, layout is viewport-only.

Rides along, mocked separately if wanted

  • Tag CHIP editor with autocomplete (replaces the comma alert; also fixes tap-a-chip-silently-deletes).
  • Photo tap → full-screen viewer (QuickLook). Dynamic Type across the page.

Must not break

  • Inline [[img]] photos · capture-quote protection (quote block above, ramble editable) · speaker-turn view for conversations (keeps its own surface for now) · transcriptUserEdited trust flag · save-now flow · polished-body editing (Phase 4) · swipe-between-memos paging.
