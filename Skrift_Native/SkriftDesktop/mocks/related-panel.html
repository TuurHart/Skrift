Design decisions — round 3 state

  • One list + a sort toggle: Date ⇄ Closest, single click, same pill idiom as Queue|Review. Default Date; remembered app-wide. Date = timeline rail (#m1), Closest = flat best-first rows (#m2). Exactly two orders, ever.

Real scale = shared 10-step 0–1 SignificanceScale (tiers Passing/Useful/Important, nil = Not rated).
  • Closeness = hover tooltip, never ambient (Tuur asked, 16 Jul): hovering a row shows "58% match · shares: …" next to the ✕ decision. Always a %, never 0.x — so it can't be misread as importance. Promote to ambient later only if device use demands it.
  • Weird embedder matches: 0.45 floor gates junk; Closest mode makes ranking judgeable; hover ✕ = "not related" hides a pairing per note (v1 hide-only).
  • Panel replaces the bottom LINKED FROM strip; collapses (#m5) with a count badge, ⌥⌘C, app-wide persisted.
  • Consent gate in-panel (#m4): phone Journal-gate mirror, one consent also unlocks Mac Journal search; local per-device index, embeddings never sync.
  • Reading-column polish = #m6, an explicit parked proposal (tags-under-chips ① / slimmer importance ② / chip icons ③); the live layout is the signed-off spec until then. Sidebar Journal → Review applied (phone parity).
  • Unlinked mentions — out (locked, later idea).
