One neighbourhood, one view. Thread and Related are the same score set (verified: threadOrder = related-above-floor + seed, oldest-first) — so listing "Related" rows AND a thread would show the same notes twice. Variant A merges them: the dated rail IS the related view, every node carries why-chips (person / #tag / term — gist-overlap heuristic, dumb v1). Importance = dot size on the rail (the journal-calendar idiom); the highest-cosine note gets a closest match flag on its date row. Click any node → that memo opens in this pane and the highlight hops. The bottom LINKED FROM strip is gone — backlinks live here now. Panel = 280 px, sidebar tint, own scroll.

Collapsed = fully gone — the reading column takes the width back; no backlinks strip returns (the panel owns connections, hidden means hidden). The toolbar toggle keeps a count badge so a folded panel still whispers that this note connects somewhere. Toggle = click or ⌥⌘C; the state is app-wide (not per note) and persists across launches. The header » chevron folds it too.

Design questions from the brief — answered

  • Does the panel collapse? Yes — toolbar toggle + header », ⌥⌘C, app-wide persisted (#m4). Collapsed keeps a count badge.
  • Does it replace the bottom LINKED FROM strip? Yes, fully — backlinks render only in the panel (both variants).
  • Empty state? The consent gate renders inside the panel — phone Journal-gate copy, one CTA, then downloading → indexing → live (#m3). One consent also unlocks Journal search on the Mac.
  • Why-chips — three kinds: person (namelink lilac) / #tag (accent) / gist term (plain). Overlap heuristic v1: shared linked names + shared tags + top shared MemoGist terms, cap 3 + "+N".
  • Importance on the rail — dot size (small / regular / large-with-glow), the journal-calendar idiom; the closest match gets a date-row flag, like FIRST MENTION / THIS NOTE.
  • The layout call (pick one): A = one dated rail carries everything (no duplication, arc always visible). B = score-ordered Related list + thread takeover with snippets (phone-familiar, richer thread, but the same notes list twice).
  • Unlinked mentions — out, parked as a later idea (locked).
