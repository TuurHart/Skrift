Build notes

Phase 2 of STANDALONE_PLAN.md — "the #1 table-stakes." One source of truth (CloudKit-mirrored SwiftData) fans out, read-only, to an Obsidian sink. These four states are the net-new UI; everything below is the real code seam each one binds to.

The export engine

  • New Services/Export/MemoExporter.swift — turns a Memo into {Markdown (Obsidian frontmatter, [[img_NNN]]→embed), plain txt, PDF, quote-card image, audio .m4a}, reusing the Phase-0 Compiler through a neutral input DTO (so the Mac and the phone compile byte-identically) + ImageMarkerReinsert. State 1's sheet is the surface; the highlighted "Export to Obsidian" routes through the publisher, the rest hit the OS share/save path.
  • ShareLink into MemoDetailView.swift:128-133 — the ⋯ confirmationDialog (already there for Copy/Delete) gains Share… + the four export verbs. "Share…" is a real ShareLink exporting the chosen representation; native iOS, no custom sheet chrome.

The Obsidian publisher (one-way, create-only)

  • New ObsidianPublisher — State 2 is its first-run. User picks the vault folder once → we persist a security-scoped bookmark (precedent already in the app: MemoSaver.swift:61, AudiobookImporter.swift:99); stale bookmarks re-prompt. We ALWAYS write under a dedicated <vault>/Skrift/ subfolder (Skrift/Voice Memos/, Skrift/Audiobook Quotes/) — Skrift owns it, never reads the rest of the vault. The privacy line in State 2 is load-bearing copy, not decoration: the bookmark is scoped to Skrift/, satisfying the repo's hard "never point at the vault" rule.
  • Per-memo export identity on Memo (additive/defaulted → clean for CloudKit + SwiftData): exportRelativePath: String?, exportedContentHash: String?, exportedAt: Date?. The sticky path means a re-export overwrites only its own file (single owner per file = no filename 2.md conflict copy). The content-hash makes re-publish idempotent — unchanged memos are skipped, so we don't churn iCloud.
  • Atomic + NSFileCoordinator writes — Obsidian / iCloud never observe a half-written .md.
  • New PublishCoordinator (peer to SyncCoordinator) fans the store out to whichever sinks are configured. State 4's batch "Export" enumerates the selection through it; the toast reports how many notes landed.

On-device name-linking — the [[Name]] in every state

  • The [[Tuur]] / [[Roksana]] links in States 1 & 4 are produced on-device by the ported Sanitiser (Phase 0, SkriftPipelineKit) running over the local names.json automatically after transcription — no extra UI for the linking itself, exactly like the Mac. So the phone's display AND its standalone Obsidian export both carry real wiki-links.
  • The phone still sends RAW to the Mac (contract spine unchanged). Linking is deterministic over the synced names DB, so the Mac re-links the same RAW with the same shared code → identical links, no double-linking, no skip-signal to miss. State 3's "Aliases & voices" row is the new alias-management surface (Phase 2, mirroring the Mac) — it syncs via the existing NamesData LWW.

Settings & significance (State 3)

  • New Settings → Export section: Default format · Obsidian vault (shows …/MyVault → Skrift/) · Include audio · the publish segmented control.
  • Significance is reframed as importance/pin (field + contract name kept for byte-compatibility). It no longer gates "does it leave the device" globally — it's a per-sink filter. CloudKit ignores it (every memo, incl. significance 0, syncs to your own devices). The "Only important (> 0)" segment is the Obsidian publish filter; the Mac transport keeps today's > 0 flag-to-send unchanged. State 1's memo shows 6/10 dots → it's in the "important" set.

Source taxonomy (State 4 glyphs)

Reuses the unified source taxonomy already shipped: 🎙 voice memo (mic.fill), ❝ audiobook quote (book glyph + italic quote lead), 📹 video (video.fill — Memo.isVideoImport), 🔗 shared URL. Same glyph + label across the row and detail; the batch bar exports whatever's selected regardless of source.

Decisions to weigh in on

  • Default publish scope — State 3 defaults to "Only important (> 0)" to keep the vault tidy and mirror today's Mac-sync gate. Should the standalone default instead be "All memos" (since CloudKit already holds everything and the Skrift/ folder is isolated)?
  • Paired-mode export ownership — when a Mac is paired it has the enhanced text, so the plan defaults Obsidian export to the Mac and turns the phone's publish off (two writers shouldn't race one note). Per-memo file ownership + content-hash make a stray double-write harmless, so this could also be a user toggle rather than an automatic switch — which do you want as the default?
  • Quote-card image export — MemoExporter can also render the Phase-6 shareable quote card. Should it appear in this sheet now (as "Export as image"), or wait for the Commonplace Book phase where the card UI properly lives?
